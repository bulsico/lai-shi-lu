生成一份"交易来时路"报告——用AI蒸馏交易历史，用中文网络流行语做毒舌但准确的交易复盘。

**用法:**
- `/trade-roast 0x123...` — 分析某个 Hyperliquid 地址
- `/trade-roast --rh stocks.csv` — 分析 Robinhood 股票/加密货币 CSV

---

## 第一步：获取并处理数据

### Hyperliquid 地址

如果参数是 `0x` 开头的地址，执行：

```bash
tsx scripts/fetch-hl.ts $ADDRESS --out data/tmp/$ADDRESS-fills.json
tsx scripts/analyze.ts --fills data/tmp/$ADDRESS-fills.json --out data/tmp/$ADDRESS-summary.json
```

然后读取 `data/tmp/$ADDRESS-summary.json`。

### Robinhood CSV

如果参数是 `--rh [文件路径]`，执行：

```bash
tsx scripts/parse-rh.ts $CSV_PATH --out data/tmp/rh-summary.json
```

然后读取 `data/tmp/rh-summary.json`。

---

## 第二步：读取市场背景参考

在生成报告前，先读取 `context/market-context.md`。这份文档总结了2023-2026年的市场规律、HL平台特性和常见行为陷阱，能帮你更准确地评价一个交易员的操作是否与当时的市场趋势顺行或逆行。

用它作为评价的背景，不要在报告里直接引用它的内容，而是将其融入判断（例如：某笔亏损发生在BTC死亡交叉期间 → 评语里自然提到"这时候做多需要勇气，但市场告诉你了"）。

---

## 语气与风格指南

**必须使用的网络流行语（自然融入，不要堆砌）：**
- 夯 = 被套牢/亏大了 ("被ENA夯了$8万")
- 寄了 = 完蛋了 ("这仓位寄了")
- 麻了 = 麻木无语 ("看到这个数字我麻了")
- 绷不住了 = 忍不住了
- 破防了 = 被戳痛点了
- 遥遥领先（讽刺用法）= "在亏损榜上遥遥领先"
- 抽象 = 操作令人困惑 ("这操作太抽象了")
- 上头了 = 冲动操作 ("那天你上头了")
- 赢麻了/亏麻了 = 达到麻木程度
- 教科书级别 = 经典案例（正反皆可）
- 草台班子 = 很混乱的操作

**禁止：**
- 不要只给正面鼓励，必须指出真实问题
- 不要捏造数据，所有数字来自 summary.json
- 不要用过于书面或正式的语言，要有生命力
- 不要超过1500字，要精炼

**数字格式：**
- 正数 P&L 用 +$X,XXX 格式
- 负数 P&L 用 -$X,XXX 格式
- 胜率用百分比 XX.X%
- 金额超过一万时用万表示（如 "$15万" 而不是 "$150,000"）
